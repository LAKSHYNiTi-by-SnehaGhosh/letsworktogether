# Contributing to Let's Work Together (LWT)

First, thank you for contributing to LWT! Our mission is to bridge the gap between education and employment, and we take our engineering and design standards very seriously.

## The Number One Rule
**Documentation First.** Before writing *any* code or installing *any* dependency, ensure you have reviewed the `docs/` directory and written or updated relevant markdown files for your feature.

## Engineering Expectations
- **Zero Placeholder Logic**: Do not use `TODO` comments for business logic. Write complete, production-ready code.
- **Enterprise Quality**: Treat this codebase as a tier-1 enterprise SaaS application. We emulate professional environments, so our code must reflect the highest professional standards.
- **TypeScript**: We use strict TypeScript. `any` is strictly prohibited.

## Workflow
1. Study the relevant documentation in `docs/`.
2. Outline your architecture or changes in an implementation plan.
3. Execute the changes following our clean architecture guidelines (`docs/3-engineering/coding_guidelines.md`).
4. Validate changes for Performance, Security, and Accessibility.
5. Submit for review.

Remember, LWT should emotionally make users feel: "I work inside a real company." Ensure your contributions reinforce this feeling.
